## 系统概述

### 设计目标

科研成果管理软件旨在为科研课题组提供一套轻量级、易部署、易维护的本地化成果管理解决方案。系统采用Python开发，无需复杂的数据库配置，支持命令行和Web两种交互方式。

### 核心设计原则

1. **简单优先**：避免过度设计，功能实现以满足核心需求为主
2. **本地优先**：数据存储本地，不依赖外部服务，保证数据安全
3. **类型安全**：使用Python dataclass和类型注解确保代码质量
4. **权限清晰**：基于角色的访问控制，权限边界明确
5. **可扩展**：模块化设计，便于后续功能扩展

### 技术选型

| 技术领域 | 技术选择 | 理由 |
|---------|---------|------|
| 编程语言 | Python 3.9+ | 跨平台、生态丰富、适合快速开发 |
| 数据存储 | JSON文件 | 轻量级、易于备份、无需数据库服务 |
| Web框架 | Python标准库 http.server | 无外部依赖、适合本地部署 |
| 前端技术 | 原生HTML/CSS/JavaScript | 无构建步骤、加载快、易维护 |
| 认证机制 | JWT + Session Cookie | 无状态、安全、易实现 |
| 密码存储 | PBKDF2-HMAC-SHA256 | NIST推荐、抗暴力破解 |

## 功能模块设计

### 成果管理模块

#### 添加成果

**输入**：
- 成果类型
- 基本信息（标题、年份、负责人等）
- 类型专属字段

**处理流程**：
1. 验证输入数据格式
2. 检查权限（需要CREATE权限）
3. 生成成果ID（根据类型前缀）
4. 创建ResearchOutput对象
5. 验证关联实体存在性（成员、项目）
6. 保存到数据库
7. 记录审计日志

**输出**：
- 成功：返回成果ID
- 失败：返回错误信息

#### 编辑成果

**权限检查**：
- PI/Admin：可编辑所有草稿状态成果
- Member：仅可编辑自己创建的草稿
- 已提交/已通过的成果需先退回

**处理流程**：
1. 加载现有成果
2. 检查编辑权限
3. 合并修改内容
4. 验证数据完整性
5. 更新时间戳
6. 保存更改
7. 记录审计日志

#### 删除成果

**权限检查**：
- PI/Admin：可删除任意成果
- Member：仅可删除自己的草稿

**处理流程**：
1. 检查删除权限
2. 检查引用关系（审核记录等）
3. 执行软删除或硬删除
4. 记录审计日志

### 审核流模块

#### 提交审核

**前置条件**：
- 成果状态为Draft或Returned
- 用户有EDIT权限

**处理流程**：
1. 验证成果信息完整性
2. 更改状态为Submitted
3. 创建ReviewRecord
4. 通知PI（如果有通知机制）
5. 记录审计日志

#### 审核通过

**权限要求**：仅PI角色

**处理流程**：
1. 验证PI权限
2. 检查状态可转换性
3. 更改状态为Approved
4. 更新ReviewRecord
5. 记录审核意见
6. 记录审计日志

#### 审核退回

**权限要求**：仅PI角色

**输入**：退回原因（必填）

**处理流程**：
1. 验证PI权限
2. 检查状态可转换性
3. 更改状态为Returned
4. 记录退回原因
5. 通知提交人
6. 记录审计日志

### 外部数据抓取模块

#### DOI抓取

**数据源**：CrossRef API

**流程**：
```
用户输入DOI
    ↓
格式验证
    ↓
调用CrossRef API
    ↓
解析响应JSON
    ↓
映射到ArticleMetadata
    ↓
返回草稿对象
```

**字段映射**：
- `title` → `title`
- `author` → `first_authors`
- `container-title` → `journal`
- `published-print.date-parts` → `publication_year`
- `DOI` → `doi`

#### PubMed抓取

**数据源**：NCBI E-utilities API

**流程**：
1. 通过ESearch获取PMID
2. 通过EFetch获取详细信息（XML格式）
3. 解析XML提取字段
4. 返回草稿对象

#### 文档解析

**支持格式**：PDF、Word、TXT

**处理流程**：
1. 读取文件内容
2. 提取纯文本
3. 正则匹配标题、年份、DOI等标识符
4. 提取摘要（前500字符）
5. 返回草稿对象

**识别规则**：
- 年份：`(19|20)\d{2}`
- DOI：`10\.\d{4,}/[^\s]+`
- PMID：`PMID:\s*(\d+)`

## 权限控制设计

### 权限判断算法

```python
def can_perform(
    actor_role: Role,
    permission: Permission,
    target_output: Optional[ResearchOutput],
    actor_member_id: Optional[str]
) -> bool:
    # 1. 角色基础权限
    if permission in ROLE_PERMISSIONS[actor_role]:
        # 2. 状态相关权限
        if target_output:
            if permission == Permission.VIEW:
                return can_view(actor_role, target_output, actor_member_id)
            elif permission == Permission.EDIT:
                return can_edit(actor_role, target_output, actor_member_id)
            # ... 其他权限检查
        return True
    return False
```

### 权限规则引擎

**规则表**：

| 规则ID | 条件 | 权限 | 结果 |
|--------|------|------|------|
| R1 | role=PI | REVIEW | ALLOW |
| R2 | role=Member AND status=Draft AND owner=self | EDIT | ALLOW |
| R3 | role=ReadOnly AND status!=Approved | VIEW | DENY |
| R4 | role=Admin | MANAGE_MEMBERS | ALLOW |

**规则优先级**：
1. 显式DENY规则优先
2. 角色基础权限
3. 上下文相关权限
4. 默认DENY

### CSRF防护

**实现机制**：
1. 登录时生成CSRF Token
2. Token存储在Session中
3. 表单提交时验证Token
4. Token有效期：8小时

**验证流程**：
```
POST请求 → 提取Token → 对比Session → 通过/拒绝
```

```json
      "title": "Example Research Paper",
      ...
    }
  ]

```

### 数据一致性保证

#### 1. 原子写入

使用**临时文件+原子重命名**策略：

```python
temp_file = f"{target_file}.tmp"
with open(temp_file, 'w') as f:
    json.dump(data, f)
os.replace(temp_file, target_file)  # 原子操作
```

#### 2. 引用完整性检查

删除成员/项目前检查是否被引用：

```python
def delete_member(member_id):
    referenced_outputs = [
        o for o in outputs
        if member_id in o.owner_member_ids
    ]
    if referenced_outputs:
        raise ValueError("Member is referenced")
    # 执行删除
```

#### 3. Schema版本管理

每个文件包含`schema_version`字段，支持未来数据迁移：

```python
if data["schema_version"] < CURRENT_VERSION:
    data = migrate(data, CURRENT_VERSION)
```

## 安全性设计

### 认证机制

**密码存储**：

```python
def hash_password(password: str, salt: bytes) -> bytes:
    return hashlib.pbkdf2_hmac(
        'sha256',
        password.encode('utf-8'),
        salt,
        iterations=180000  # OWASP推荐
    )
```

**会话管理**：
- Session ID：32字节随机数
- 存储：内存字典（服务重启失效）
- TTL：8小时
- Cookie：HttpOnly + SameSite=Strict

### 输入验证

**通用规则**：
1. 所有用户输入必须验证
2. 长度限制：标题≤500字符，摘要≤5000字符
3. 特殊字符转义（HTML注入防护）
4. SQL注入：不适用（无SQL数据库）

**ID格式验证**：
```python
# 成果ID：前缀-年份-序号
pattern = r'^(LW|ZL|RZ|HY|XM|SJ)-\d{4}-\d{3}$'
```

### 文件上传安全

**限制**：
- 最大文件大小：50MB
- 允许的文件类型：PDF、DOC、DOCX、TXT
- MIME类型验证
- 文件名长度限制：255字符

**存储位置**：
```
data/local/uploads/<year>/<month>/<random_filename>
```

### 审计日志

**记录内容**：
- 操作时间戳
- 操作用户
- 操作类型（CREATE/UPDATE/DELETE/APPROVE等）
- 目标实体ID
- 操作前后的数据快照（可选）

**日志格式**：
```json
{
  "timestamp": "2026-07-01T12:34:56Z",
  "actor_id": "alice",
  "action": "approve_output",
  "target_id": "LW-2026-001",
  "details": {"status_change": "submitted -> approved"}
}
```
