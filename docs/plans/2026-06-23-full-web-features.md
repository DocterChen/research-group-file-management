# 课题组科研成果管理系统完整功能实施计划

**制定日期**: 2026-06-23  
**计划状态**: 待执行  
**风险等级**: 高（涉及认证、文件上传、数据库集成、外部API）

## 目标

将当前的本地CLI+简单Web UI升级为功能完整的课题组科研成果管理系统，支持专业认证、中文界面、文件上传、Excel导出、外部数据抓取和全文检索。

## 技术选型与理由

### Web框架：保持现有架构 + 增强

**选择**: 保持 Python http.server + 渐进增强
**理由**:
- 当前代码已实现基础认证和UI渲染
- 避免大规模重写，降低风险
- 可以逐步集成第三方库

**替代方案**: FastAPI/Flask
- **不采用理由**: 需要完全重构，风险太高；当前架构已满足需求

### 认证系统：JWT Token

**选择**: PyJWT + 改进的密码哈希
**理由**:
- 轻量级，与现有代码兼容
- 支持会话管理和过期控制
- 可以与前端无状态通信

**实现方案**:
- 保留现有的 LocalAuthStore 和 SessionStore
- 增加 JWT token 生成和验证
- 支持 refresh token

### 文件上传：分块上传

**选择**: 自实现分块上传 + 文件元数据管理
**理由**:
- 支持大文件（>100MB）
- 支持断点续传
- 与现有 repository 模式一致

**技术要点**:
- 使用 Range 请求头
- 临时目录存储分块
- 完成后合并并移动到最终位置
- 记录附件元数据到 attachments.json

### Excel导出：openpyxl

**选择**: openpyxl
**理由**:
- 纯Python实现，无需外部依赖
- 支持样式、公式、图表
- 与现有CSV导出逻辑兼容

**功能**:
- 多Sheet导出（成果、成员、项目）
- 样式格式化（标题、边框、颜色）
- 筛选器和冻结窗格
- 统计汇总Sheet

### 数据抓取：requests + 专业API

**选择**: 
- DOI: CrossRef API
- PubMed: NCBI E-utilities
- 专利: 国知局或Google Patents API

**理由**:
- 官方API，数据质量高
- 免费配额充足
- 返回结构化数据

**限制**:
- 需要遵守API rate limit
- 某些数据需要API key
- 网络依赖

### 全文检索：Whoosh

**选择**: Whoosh
**理由**:
- 纯Python实现，无需外部服务
- 支持中文分词（通过jieba）
- 索引文件本地存储

**替代方案**: Elasticsearch
- **不采用理由**: 需要独立服务，部署复杂，资源占用大

**索引内容**:
- 成果标题、摘要、关键词
- 作者、期刊名称
- 附件文本内容（PDF转文本）

### 多人编辑：乐观锁 + 版本号

**选择**: 
- 每个实体增加 version 字段
- 更新时校验版本号
- 冲突时提示用户

**理由**:
- 简单可靠
- 无需引入复杂的协同编辑框架
- 适合小团队场景

**不支持**:
- 实时协同编辑（类似Google Docs）
- WebSocket推送

## 风险与应对

### 风险1：文件上传并发冲突

**影响**: 文件损坏或丢失
**概率**: 中
**应对**:
- 使用文件锁
- 唯一文件名（UUID）
- 临时目录隔离

### 风险2：外部API不可用

**影响**: 数据抓取失败
**概率**: 中
**应对**:
- 降级到手动输入
- 缓存API响应
- 支持离线模式

### 风险3：全文检索性能

**影响**: 搜索缓慢
**概率**: 低
**应对**:
- 限制索引内容大小
- 异步索引更新
- 分页结果

### 风险4：数据模型变更

**影响**: 现有数据不兼容
**概率**: 高
**应对**:
- 数据迁移脚本
- schema_version 校验
- 备份和回滚机制
