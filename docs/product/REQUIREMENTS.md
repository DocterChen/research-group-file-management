# 科研成果管理 MVP 需求说明

## What

本 MVP 用于课题组内部以本地 CLI 与本地 Web UI 管理科研成果与基础审核流，优先验证统一领域模型、角色权限和可追溯数据结构。

## 当前覆盖范围

- 成果类型：文章、专利、软件著作权、会议成果、项目材料、数据代码。
- 实体：成果、成员、项目、审核记录、审计日志。
- 操作：新增成员、新增项目、新增成果、提交审核、审核通过、查看列表、查看详情、汇总统计、CSV/Excel 导出。
- Web UI：支持首次初始化工作区名称、中文成果类别展示、按类别自动生成成果编号、负责人/参与人勾选或手动输入。
- 存储：本地 JSON 多文件结构。

## 目标用户

- `pi`：查看全组成果、执行审核、查看统计与导出。
- `admin`：拥有全部权限，协助维护成果、账号审核和审核流。
- `member`：维护自己负责的成果并提交审核。
- `readonly`：只读查看。

## MVP 非目标

- 真实附件上传与下载
- 外部数据库

## 验收标准

- 可以通过 CLI 完成成员、项目和文章成果的完整录入。
- 普通成员只能查看、维护和提交自己负责的成果，不能审批或导出全量数据。
- PI 可以审批已提交成果。
- 所有成果、审核记录和审计日志都可持久化到本地 JSON。
- 可以输出汇总统计和 CSV 文件。
