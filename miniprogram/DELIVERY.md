## 使用说明

### 启动前准备

1. **后端配置**（在项目根目录）：
   ```bash
   # 配置微信小程序凭证
   vim .env  # 或编辑 config.yaml
   
   # 添加以下内容
   WECHAT_MINIPROGRAM_APPID=your_appid
   WECHAT_MINIPROGRAM_SECRET=your_secret
   
   # 启动后端 API
   python api_server.py
   ```

2. **小程序配置**：
   ```javascript
   // 编辑 miniprogram/app.js
   globalData: {
     apiBase: 'http://localhost:8080/api/v1',  // 开发环境
     // apiBase: 'https://your-domain.com/api/v1',  // 生产环境
   }
   ```

3. **打开小程序**：
   - 打开微信开发者工具
   - 导入项目 → 选择 `miniprogram` 目录
   - AppID：使用测试号或正式 AppID
   - 开启"不校验合法域名"（开发阶段）

### 开发流程

1. **本地调试**：
   - 启动后端服务（确保运行在 http://localhost:8080）
   - 打开微信开发者工具
   - 点击"编译"运行小程序
   - 使用"预览"功能在手机上测试

2. **真机调试**：
   - 使用内网穿透工具（natapp/frp）
   - 修改 `apiBase` 为穿透后的 HTTPS 地址
   - 点击"预览" → 扫码 → 手机调试

3. **生产部署**：
   - 后端部署到云服务器（配置 HTTPS）
   - 修改 `apiBase` 为生产环境地址
   - 在微信公众平台配置服务器域名白名单
   - 上传代码 → 提交审核

## 注意事项

### 图标资源
当前 TabBar 配置引用了图标路径，但图标文件未提供。两种解决方案：

1. **临时方案**：修改 `app.json`，移除 `iconPath` 和 `selectedIconPath`，使用纯文字 TabBar
2. **完整方案**：根据 `assets/icons/README.md` 准备图标资源

### API 接口
小程序已完整集成后端 API 接口，但需要确保后端已实现对应的端点（参考 `api_extensions.py`）。

### 域名配置
- 开发阶段：关闭域名校验即可使用 HTTP
- 生产阶段：必须配置 HTTPS 和域名白名单

### 微信开放平台
如需获取 UnionID（跨平台统一身份），需要：
1. 注册微信开放平台账号
2. 绑定小程序和公众号到同一主体
3. 完成开发者资质认证
