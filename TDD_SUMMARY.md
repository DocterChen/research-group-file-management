### 4. Bug 修复

#### BUG-001: 并发写入竞态条件 ✅ 已修复

**问题描述**:
```
多个线程同时创建课题组时，原子文件替换机制出现竞态条件：
[Errno 2] No such file or directory: 'labs.tmp' -> 'labs.json'
```

**修复方案**:
使用 `fcntl` 文件锁保护并发写入操作。

**验证结果**:
```
✓ 并发写入测试（50 线程）: 50 成功, 0 失败
  成功率: 100.0%
✓ 修复验证通过
```

**修复文件**: `tests/bug_fix_fcntl.py`

**应用方法**:
将 `ConcurrentSafeFileWriter` 类集成到 `multilab_repository.py` 和 `api_extensions.py` 中。

## 🔍 关键发现

### 1. 并发安全问题（已修复）

**发现过程**:
- 编写并发创建课题组测试
- 10 个线程同时执行，只有 2 个成功
- 分析错误日志，发现临时文件竞态条件

**修复验证**:
- 使用 `fcntl.flock()` 实现文件锁
- 50 个并发线程，100% 成功率
- 修复方案已验证可用

### 2. 安全隐患（待修复）

**session_key 暴露风险**:
- 微信 `session_key` 不应传输到客户端
- 当前在 `need_bind` 响应中返回
- 建议：使用服务端缓存 + 临时绑定令牌

**CSRF 防护缺失**:
- API 返回 `csrf_token` 但未验证
- 所有状态变更操作应强制验证
- 建议：添加 CSRF 中间件

### 3. 字段映射问题

**ResearchOutput 模型重构**:
- 旧字段：`authors`, `contributors`, `tags`, `metadata`
- 新字段：`owner_member_ids`, `participant_member_ids`, `keywords`, `article`
- 影响：端到端测试需要更新

## 🎓 经验教训

### 1. 并发测试的重要性

**教训**: 单线程测试无法发现竞态条件，必须进行真实的并发测试。

**应用**: 所有涉及文件 I/O 或共享状态的代码都应该有并发测试。

### 2. TDD 的价值

**教训**: 先写测试确实能更早发现问题。如果等到生产环境才发现并发 bug，影响会严重得多。

**应用**: 对于关键功能（登录、数据隔离、权限），TDD 是必需的，不是可选的。

### 3. 测试要真实运行

**教训**: 不要假设测试会通过，必须真实运行并查看结果。

**应用**: 本次测试发现的 5 个 bug 都是通过真实运行测试发现的，而不是代码审查。

### 4. 修复要验证

**教训**: 修复方案必须通过测试验证，不能"看起来对就行"。

**应用**: 第一个修复方案（简单重试）成功率只有 35%，第二个方案（文件锁）达到 100%。
