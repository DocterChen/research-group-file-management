# Changelog

本文件记录项目重要变更，格式遵循 Keep a Changelog，并优先维护 `[Unreleased]`。

## [Unreleased]

### Added（新增）

- **用户注销功能**：新增用户自助申请注销账号功能
  - 用户可在"账号设置"页面申请注销账号
  - 注销申请需要管理员审批
  - 管理员在"账号审核"页面可批准或拒绝注销申请
  - 批准后用户账号将被永久删除
  - 申请后在账号设置页显示等待审批状态
  - 导航栏新增"账号设置"入口
- **账号设置页面**：新增独立的账号设置页面，显示用户基本信息和账号管理选项
- **账号审核页面增强**：账号审核页面合并展示注册审核和注销审核两个独立表格
  - 注册审核：审核新用户注册申请
  - 注销审核：审核用户注销申请，支持批准和拒绝操作
- **CSRF 防护**：为所有表单添加 CSRF 令牌验证，防止跨站请求伪造攻击。每个会话生成独立的 CSRF 令牌，提升安全性。
- **表单大小限制**：增加表单提交大小限制（10MB）和文件上传大小限制（50MB），防止恶意大文件攻击。
- **成果归档功能**：增加 `archive_output` 方法，支持将已通过审核的成果归档。管理员和 PI 可在成果详情页和审核工作台使用归档功能。
- **成果列表搜索功能**：成果管理页面增加搜索框，支持按标题或编号搜索成果。
- **成果列表筛选功能**：增加成果类型和审核状态筛选器，方便快速定位特定成果。
- **成果列表分页功能**：支持分页显示成果列表，每页默认显示 20 条记录，提升大数据量场景下的浏览体验。
- **成果列表操作列**：在成果列表中增加操作列，每条记录可直接点击"查看"按钮进入详情页。
- **成果详情页删除按钮**：在成果详情页为有删除权限的用户显示删除按钮，支持直接删除成果。
- **退回和归档入口优化**：审核工作台和成果详情页增加退回、归档按钮，完善审核流程闭环。

### Changed（变更）

- **项目管理负责人选择框UI优化**：优化项目管理表单中负责人选择框样式
  - 增加内边距（10px）和圆角（8px）
  - 添加平滑过渡动画效果
  - 优化光标样式，提升交互体验
  - 保持与其他表单元素风格一致
- **导航栏待处理数量统计优化**：账号审核入口的徽章数量现在包含注册审核和注销审核的总和
- **WebUser数据模型扩展**：在 `WebUser` 类中新增 `deletion_requested_at` 和 `deletion_approved_by` 字段，支持注销流程追踪
- **数据保存优化**：JSON 数据保存改用临时文件+原子替换策略，先写入临时文件再替换目标文件，避免写入过程中断导致数据损坏。应用于 `repository.py` 的 `_save_entities` 方法和 `web.py` 的 `_save_users` 方法。
- **移除默认管理员自动创建**：首次启动不再自动创建默认管理员账号，必须通过 `/setup` 页面手动设置工作区和管理员信息，提升安全性。
- **DOI/PubMed 抓取错误提示优化**：改进外部数据抓取的错误处理，失败时显示明确的错误信息（包括具体失败原因和建议），不再静默失败。
- **窄屏侧边栏优化**：改进响应式布局，窄屏（<1080px）下侧边栏不再保持 sticky 定位，避免遮挡内容，改为正常流式布局。
- **手动输入优先级调整**：成果和项目表单中，手动输入框移到选择列表上方，优化操作流程。
- **审核状态按钮优化**：成果详情页根据当前审核状态智能显示可用操作按钮（草稿/退回状态显示"提交审核"，已提交状态显示"通过审核"和"退回"，已通过状态显示"归档"）。
- **Codex 执行规则优化**：在 `AGENTS.md` 中优化 Codex 执行规则章节
  - 移除冗余的"自主决策与完美执行"条目
  - 整合相关规则，保持章节简洁清晰
  - 保留核心要求：全程按 awesome-code 规范、自主决策、保证程序完美执行、执行规则写入规范

### Fixed（修复）

- **requests 库依赖**：确认 requests 库已安装，DOI/PubMed 抓取功能可正常使用。

### Security（安全）

- **CSRF 防护**：防止跨站请求伪造攻击。
- **表单大小限制**：防止恶意大文件上传导致的拒绝服务攻击。
- **原子文件替换**：防止数据写入中断导致的数据损坏。

### 审核工作台待审核数量提示**：在审核工作台页面标题处显示待审核成果数量徽章，侧边栏导航的"审核工作台"和"账号审核"链接也显示相应的待处理数量徽章，方便管理员快速了解待办事项。
- **执行规则写入规范**：在 `AGENTS.md` 的 Codex 执行规则章节新增条目，明确当用户要求"把这个规则写入到 AGENTS.md"时的处理流程。

### Changed（变更）

- **示例数据匿名化**：将 `examples/research_outputs.sample.json` 中的具体人名、机构名等敏感信息替换为"某某"、"某某教授"等匿名占位符，以保护隐私并符合示例数据规范。
- **界面设计系统现代化优化**：全面升级 Web UI 的 CSS 设计系统，包括：
  - 更新配色方案：采用更现代的蓝色系主题（sky blue），提升专业感和可读性
  - 优化间距和圆角：使用更合理的间距层级（8px、12px、16px 等）和圆角值
  - 改进交互反馈：为按钮、链接、表单控件添加流畅的过渡动效和悬停状态
  - 增强视觉层级：优化徽章、表格、卡片的对比度和阴影效果
  - 统一字体排版：改进字号、字重和行高，提升可读性
  - 优化状态徽章：为不同审核状态（草稿、已提交、已通过、已退回）使用语义化配色

### Fixed（修复）

- **修复 cgi 模块依赖问题**：将文件上传功能从已废弃的 `cgi.FieldStorage` 迁移到标准库的手动 multipart 解析，确保 Python 3.13+ 兼容性
- **修复专利拉取降级逻辑**：改进 `fetch_patent_by_number` 异常处理，确保 PatentsView API 不可用时返回降级数据而非 None
- **修复审核后跳转逻辑**：成果详情页的"通过审核"按钮现在会根据用户角色智能跳转（管理员/PI 跳转到审核工作台，普通用户跳转到成果列表）
- **统一登录错误提示文案**：将"姓名或密码错误"改为"账号或密码错误"，与登录表单标签保持一致

### Changed（变更）

- **强化 Codex 执行规则**：在 `AGENTS.md` 中明确以下要求
  - 全程按 awesome-code skill 规范执行复杂开发任务
  - 自主决策不询问用户：遇到疑问或更优方案时自行选择最保守且可验证的实现
  - 保证程序完美执行：最终交付前必须完成可运行验证，所有问题必须解决

### Added（新增）

- **文档上传自动识别草稿**：外部数据抓取页新增文档上传入口，支持对 PDF、Word 和纯文本进行基础字段识别并预填成果表单。

- **验证码登录与审核工作台增强**：登录页新增数字验证码，管理员可进入独立审核工作台集中处理已提交成果，并支持成果退回。

- **成果草稿保存**：成果添加与编辑页新增“保存草稿”入口，支持未提交成果反复修改后再提交审核。

- **成员管理员提权按钮**：成员管理页与成员详情页新增“设为管理员”按钮，管理员/PI 可直接提权并同步成员角色。

- **首次初始化通用字段**：工作区首次初始化时可同时设置组织名称与工作台副标题，去除对特定课题组名称的依赖。

- **Codex 执行规则补充**：在 `AGENTS.md` 中明确要求当用户指示“自己决定，不要问我”时，优先采用最保守且可验证的实现方式。

- **成果管理权限与通用化配置增强**：新增按成果类别自动生成编号、首次初始化工作区名称设置、成果/项目负责人和参与人手动输入能力，支持未建档成员或外部合作者参与记录。

- **自助注册与审核流程**：新增注册页面，成员可用姓名和密码提交注册申请；申请默认为待审核状态，管理员/PI 可在“账号审核”页面通过申请，通过后自动创建普通成员档案。

- **DOI 抓取示例与解析增强**：外部数据抓取页默认使用 `10.3390/biom12060824` 作为 DOI 示例；CrossRef 抓取逻辑支持 DOI URL/`doi:` 前缀规范化、URL 编码、多日期字段解析和摘要基础清理。

- **版本号管理规范**：在 `AGENTS.md` 中新增完整的版本号管理章节
  - 明确配置文件作为版本号的唯一真理来源（Single Source of Truth）
  - 定义版本号命名规则，遵循语义化版本规范（SemVer）
  - 规定版本同步机制：配置文件 → 项目文档 → CHANGELOG.md
  - 说明版本号更新时机：bug 修复递增修订号，新功能递增次版本号，破坏性变更递增主版本号
  - 提供版本号检查命令示例

- **登录页实验室化重设计**：登录/初始化页面加入实验室入口元素、成员/PI 分区说明和更专业的视觉层次。

- **成员登录区分说明**：登录页明确展示普通成员与 PI / 管理员的进入后职责差异，帮助用户区分入口。

- **外部数据抓取 Web UI**：新增 DOI / PubMed / 专利号抓取页面，并可从成果编辑页进入。

- **完整Web UI管理界面**：实现成员、项目、成果的完整CRUD操作（添加、编辑、删除）
  - 成员管理：添加/编辑/删除成员，支持角色选择和邮箱配置
  - 项目管理：添加/编辑/删除项目，支持多负责人选择和资助来源
  - 成果管理：添加/编辑/删除成果，支持文章类型动态表单
  - 引用完整性检查：删除前自动检查成员/项目是否被成果引用
  - 表单验证和友好错误提示

- **Excel高级格式导出**：使用openpyxl实现专业Excel报告
  - 汇总统计工作表：基础统计、类型分布、状态分布
  - 成果清单工作表：包含所有成果字段、样式格式化、自动筛选
  - 成员清单工作表：完整成员信息
  - 项目清单工作表：项目详情和资助信息
  - CLI命令：`export excel --output <path>`
  - Web UI导出：仪表盘”导出Excel”按钮

- **外部数据自动抓取**：支持DOI和PubMed数据库
  - DOI抓取：通过CrossRef API自动填充文章信息（标题、作者、期刊、年份等）
  - PubMed抓取：通过NCBI E-utilities API获取文章元数据
  - 专利抓取：新增公开专利号/申请号抓取入口并预填专利字段
  - 新增模块：`src/lab_literature_manager/data_fetcher.py`
  - 支持命令行和编程接口调用

- **全面中文化**：所有界面文本改为简体中文
  - 登录页面、设置页面完全中文化
  - 仪表盘、成员、项目、成果页面中文化
  - 表单标签、按钮、错误提示中文化
  - 图表标题和统计标签中文化

- **Repository层完整CRUD方法**：
  - `update_member()`, `delete_member()`: 成员更新和删除
  - `update_project()`, `delete_project()`: 项目更新和删除
  - `update_output()`, `delete_output()`: 成果更新和删除（带权限检查）
  - 引用完整性检查：防止删除被引用的成员/项目

- 更新 `AGENTS.md` 代码优化规则：明确要求使用awesome-code规范、自主决策、确保程序完美运行
- 同步 `CLAUDE.md` 的执行约束，保证 Claude Code 与 Codex 指令一致
- 新增依赖：openpyxl（Excel导出）、PyJWT（认证增强）、requests（数据抓取）
